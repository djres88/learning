1. INTRO: TEXT LEVELS

That header was made using an HTML element -- specifically, <h1> (the "opening tag") and </h1> (the "closing tag"). Most HTML elements have an opening tag and a closing tag, with the difference being the presence of a "/" in the closing tag. In addition to h1 headers, there are other "levels" of header elements: h2, h3, up to h6 headers.

There are paragraphs, too: p elements are the preferred method for normal-sized paragraph text on websites. You can create a p element like so:

<p>I'm a p tag!<p>

You can also escape the code in html by bracketing text between <!-- and -->. This allows you to make comments on the code, without affecting the code itself.

NOTE: You can escape the code in other ways, too, which comes in handy for projects such as this website. In html, the characters < and & (among others) refer to code. If you want to enter these and other characters as TEXT, you need to do some special work. See http://text-symbols.com/html/ for more info on how to write code as text.

3. CSS

When you entered <h2 style="color: red">CatPhotoApp</h2> , you were giving that individual h2 element an inline style. Often the better way is to use CSS ("Cascading Style Sheets"). Besides color, there are hundreds of CSS properties you can use to change the way an element looks on your page.

To use CSS, first create a style element: <style></style>.

Inside that style element, you can create a CSS selector for all h2 elements. For example, if you wanted all h2 elements to be red, you would write something like this in the style section:

 <style>
   h2 {
    color: red;
   }
 </style>.

Note the opening and closing curly braces ({ and }) around each element's style. Also be sure to add the semicolon to the end of each of your element's styles.


4. STYLE CLASSES

Classes are reusable styles that can be added to HTML elements. Here's an example CSS class declaration:

 <style>
  .blue-text {
   color: blue;
  }

We've created a CSS class called blue-text within the <style> tag. Note the presence of a "." preceding the class name. Once you declare a class, you can apply it to an html element. For example, here we apply .blue-text to the h2 element:

<h2 class="blue-text">CatPhotoApp</h2>.

WHY IS THIS USEFUL? Instead of applying a single style to every h2 element — or dictating the styles for each individual header — you can "save" a class (a style setting) and call it as needed. Think of these classes as style templates.

Note that you can apply multiple classes to an element like so:

<p class = "class1 class2"> text goes here </p>. (all within THE SAME quotes)

5. FONT-SIZE & FONT-TYPE

You can also change the FONT SIZE using font-size and px (for example, h1 { font-size: 30px; }). You can change the FONT TYPE by using the font-family property. For example, here's what you would do if you wanted to set a style class "classy" with font size 16px and font type Sigmar One:

<style>
 .classy {
  font-size: 28px;
  font-family: 'Sigmar One', cursive;
  background-color: yellow;
 }

Now we could apply the class to an h2 header. For example, <h2 class="classy">Where's my Taxi?</h2> will become:

Where's my Taxi?


6. IMPORTING FONTS

We can make calls to Google to load fonts into our HTML; the font we just used, Sigmar One, is an example. To add a font, copy the code snippet from Google and paste it into your code editor, above the styles (see the source code to this doc for an example). If we want to add, say, the Lobser font, we can write:

<link href="http://fonts.googleapis.com/css?family=Lobster" rel="stylesheet" type="text/css">.

Now you can set Lobster as a font-family value on your h2 (or any other) html element:

h2 {
 font-family: Lobster;
}

You can also specify how fonts should "degrade" if a font isn't available. Think of these as the "backup plan" fonts. For example, if you wanted an element to use the Helvetica font, but also degrade to the Sans-Serif font when Helvetica wasn't available, you could use this CSS style:

{ font-family: Helvetica, Sans-Serif; }.

For a complete list of fonts, please see the links at the bottom of the page (go to the link for Google Fonts).

9. ANCHORS

You can link to another website using the anchor tag, <a>:

Text Symbols: <a href = "http://text-symbols.com/html/"> HTML Text Symbols </a>.
Text Symbols: HTML Text Symbols

The first part of the code creates an open 'a' tag in which 'href' is set equal to a weblink; the weblink can show any information as long as the link has a valid URL. In the above example, we link to the site http://text-symbols.com/html/. The second part of the code sets the display text for the link. Above, the display text is "Text Symbols."

a. Dead Links

When developing a site, there are times when you won't yet know where your display text will link. That's where dead links (#) are useful. To use a dead link, just set href = # within your opening a-tag:

<<a href = #> HTML Text Symbols </a>

b. Turn Images into Links

The "a" tag also lets you turn images into links. Recall how we set the display text in the above example. Here, we replace the display text with a display image like so:

<a href="#"><img src="https://www.youtube.com/watch?v=VM5-xFenaZI"/></a> .

Note the continued presence of the "#" in the code above. While the image of the cat will display on the screen, the # means that the image will not yet link anywhere. To create a link, we will need to replace the "#" with a valid URL:

<a href="https://www.youtube.com/watch?v=VM5-xFenaZI"><img src="http://i.ytimg.com/vi/_W_g7_KJP_o/0.jpg"/></a>

The above code will display a still shot from Joshua Logan's Oscar-winning film, Paint Your Wagon (1997), along with a link to its most famous clip:



c. Alt Text for Images

If (when) browsers fail to load an image, they will display the alt text. Search engines care about alt text, too. In short, every image should have alt text.

Alt text should immediately follow your image source. Here's an example:

<a href="URL"><img src="IMAGE URL" alt="your alt text">

12. <DIV>

Notice how this site alternates in background color? That's all thanks to Div, a/k/a the division element.

Div is a very common (possibly the most common) HTML tag. A "general purpose" tag, you use div to group elements together, a technique also known as "nesting." Div's ability to nest various elements together proves extremely useful for applying your CSS -- recall that CSS contains your various styles and formatting -- to particular sections of the text.

For example, let's say that you want to give a particualr section of text a light blue background. To do that, you can insert the class for a lightblue background within an opening div tag, then close the div tag around the section you wish to receive the background. (NOTE: You must first define the class in CSS.) Here's how this would look in code:

<style>
 .blue-background {
  background-color: lightblue;
 }
</style>
<div class="blue-background">
 <p>This text will show up in blue
 This will too!</p> </div>

13. SET ELEMENT ID

Element IDs will be useful when we get into jQuery. You can define an element ID immediately after the tag:

<h2 id="best-header">

Every element should have a unique ID -- that is, do not use the same ID for two different elements. Every time you use the same ID twice, a puppy will be killed.


I would prefer not to die.

You can style an ID using CSS, just like you would a class. The only difference is that your CSS is referencing a unique element -- not the other way around.

For example, a class might be defined as
.classone {
 font-size:80;
}

By contrast, the style section uses the # symbol to reference an ID within the body. We could create a style for the "best-header" example (from before) by writing the following code in the style section:
#best-header {
  font-style:bold;
  font-size:30;
  background-color:black;
  color:white;
}



14. PADDING, BORDER & MARGIN

All HTML elements are essentially little rectangles. Three important properties control the space surrounding your little rectangles: Padding, Border, and Margin.

  1. Padding: Controls the amount of space between an element ("content", in the visual below) and its border.
  2. Margin: Controls the amount of space between an element's border and surrounding elements.
  3. Border:
Source: w3schools.com.

Padding, margin, and border can all be manipulated with the CSS. You can create the class for an element -- or reference the element's ID, if it has one -- and assign characteristics to manage the reference element's size. For example, if we wanted to create a purple box with padding 40px and margin 10px, we would write the following in our style section:
.purple-box {
 background-color:purple;
 padding:40px;
 margin:10px;
}
 We could then apply the class (using <div class = "classname">) to a set of elements in the body. Below, we apply class purple-box to the text "Hey, look at me! I'm a purple box!"

Hey, look at me! I'm a purple box!

b. Padding: Top to Bottom, Side to Side

CSS allows you to control the padding of an element on all four sides with padding-top, padding-right, padding-bottom, and padding-left properties.

.purple-box {
 background-color:red;
 padding-top: 40px;
 padding-right: 20px;
 padding-bottom: 20px;
 padding-left: 40px;
}

This is useful. You can use these tools to align text (or other content within an element) anywhere on the screen -- right, bottom, left, etc.

c. Margin Calls

Just as you use padding to maneuver content within an element, you can use margin to (a) change the SIZE of the element and (b) to MOVE the element around the page. For example, let's say you wanted to make the purple box smaller, thinner, and closer to the right of the page:
.purple-box-two {  background-color: purple;
 padding:15px
 margin-top:-100px;
 margin-right:0px;
 margin-bottom:20px;
 margin-left:500px;
}

I'm a purple box 2.


There! Notice, too, how purple box 2 is level with the code to its left. This is the effect of negative margin, and it's the first step toward understanding how to layer elements on a page. In the code, we added negative margin for "margin-top" in order to move the box farther up on the page. When you add negative margin, you can move an element closer to, beside, or even on top of its neighboring elements.

You might have noticed that purple box 2 wasn't quite flush the end of the page, either, despite the 0px margin. That's easily solved by making the margin-right negative -- something like -10px should do the trick.

d. Specify Margins & Padding on One Line

You can specify margins and padding using "clockwise notation," starting from the top. For the above example (i.e. our old friend purple-box-two), you could specify the margins as:
margin: -100px 0px 20px 500px
So the top margin is -100; right is 0; bottom is 20; and left is 500.

As you might expect, padding can work in the same way:
padding: -100px 0px 20px 500px



15. PRIORITIZING STYLES

Sometimes your HTML elements will receive multiple styles that conflict with one another. You can always override the default <body> CSS by assigning your tag a new class.

You can apply multiple classes to a single tag like so: class="class1 class2". Which class wins out depends on their order in your CSS.

a. Override Class Using IDs

You can override class using IDs. Recall that IDs are assigned first within a opening tag, like so

<h1 id="orange-text" class="pink-text blue-text">Hello World!</h1>

Hello World!

As long as the id "#orange-text" exists in the CSS, it will take precedence over both classes ".pink-text" and "blue-text."

b. Override Class Using Inline Styles

It's just as easy to override class using an inline style:
<h1 style="color:orange" class="pink-text blue-text">Hello World!</h1>

Hello World!

c. Override Using Important

In many situations, you will use CSS libraries. These may accidentally override your own CSS. So when you absolutely need to be sure that an element has specific CSS, you can use !important. To override both ID and Class, use !important inside the style declaration: color:pink !important



16. HEX CODES & RGB IN BRIEF

You'll probably find it easier to use this list of font colors , but here's some info on HEX codes.

Hex codes are the six-digit color codes that CSS uses to color its, well, world. Or whatever. Hex code follows the red-green-blue ("rgb") format. The first two digits of hex code represent the amount of red in the color. The third and fourth digit represent the amount of green. The fifth and sixth represent the amount of blue. That's useful knowledge if you'd like to play around with your own colors.

Alternatively, you can set colors using RGB values. Here's how:
<style>
 body {
  background-color: rgb(0,0,0);
 }
</style>
